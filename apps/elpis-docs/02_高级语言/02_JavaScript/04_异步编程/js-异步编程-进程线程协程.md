每一个应用程序有很多进程但至少有一个进程，进程之间相互独立，即使通信 也需要双方同意。

每个应用程序至少有一个进程，每个进程至少有一个线程。浏览器是一个 多进程多线程的应用程序，JS 是一个单线程语言是因为 JS 只能被浏览器的渲染主线程执行，而渲染主线程只有一个，所以 JS 是单线程语言。 
● 线程 
一个进程至少有一个线程 所以在进程开启后 自动创建一个线程来运行代码 这就是渲染主线程 (主线程结束整个程序就结束了。 如果程序需要同时执行多块代码 主线程就会启动更多的线程来执行代码 比如 网络线程  所以一个进程可以包含多个线程 )

## 协程

在 JavaScript 中，协程（Coroutine）是通过生成器函数实现的。虽然 ES6 规范中没有直接称为 "coroutine" 的特性，但生成器函数具备了协程的核心能力：可以暂停执行并恢复执行，同时保持函数内部状态。

核心定义
协程是一种可以暂停执行并恢复执行的函数（或子程序），保持其局部状态。你可以把它看作比线程更轻量级的“微线程”，但完全由程序员在用户空间控制切换，而不是操作系统调度。



进程 线程 由 操作系统调度
协程由 程序员调度
