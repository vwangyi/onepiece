# 事件循环



```js
● 事件循环
  ○ 是什么
    ■ 事件循环是 为了解决单线程语言 如何调度任务 实现异步编程的核心机制
    ■ JS 把任务分为同步任务和异步任务，异步任务分为微任务宏任务。
    ■ 执行顺序是 先同步后异步 先微任务后宏任务，执行栈每次执行宏任务前 都会轮询微任务队列 清空微任务队列后再执行一个宏任务
  ○ 优缺点
    ■ 好处是 非阻塞。将异步任务交给其他线程处理，不会阻塞主线程。
    ■ 缺点是 长任务仍然会阻塞
  ○ 讲应用
    ■ 平时开发中 应该尽量避免长任务
    ■ 常见的微任务：Promise.then catch finally
    ■ 常见宏任务：setTimeout setInterval 交互事件
```

```js 
常见的宏任务有 setTimeout、setInterval、I/O、UI渲染、事件回调等
常见的微任务有 Promise.then/catch/finally、
MutationObserver、queueMicrotask，以及在Node.js里的 process.nextTick。”

详细版：当主线程执行一段 JS 代码，遇到同步任务立即执行，遇到异步任务交给其他线程
（当异步任务满足触发条件就会进入任务队列进行排队等待主线程执行），
主线程执行完同步任务后，每次执行宏任务前都会先清空微任务队列再执行宏任务
（最新消息：宏任务中又细分了很多任务队列 比如交互队列比延时队列优先级高）

把该带出来的点 都带出来  >>> 单线程 异步编程 任务队列排队 微任务举例 宏任务举例 
● 是什么：事件循环是 为了解决单线程语言 如何调度任务 实现异步编程的核心机制。 
● 具体实现就是 排队： JS 把任务分为同步任务和异步任务，异步任务分为微任务宏任务，常见的微任务有Promise.resolve().then() ，常见的宏任务有 setTimeout() setInterval()。
  ○ 渲染任务 比如修改 xx 触发重排重绘 修改 transfrom opcity 不会触发重排重绘 
  ○ 在执行 JS 同步任务就会产生异步任务：异步任务有 渲染任务 微任务 宏任务
● 执行顺序： 执行顺序是 先同步后异步 先微任务后宏任务，执行栈每次执行宏任务前 
都会轮询微任务队列 清空微任务队列后再执行一个宏任务 （因为每次执行任务都可能产生新的微任务）
● 好处：避免了主线程的阻塞，保证了用户界面的流畅性和程序的响应性。
```

## 事件循环的一轮
```js
事件循环的一轮
事件循环的一轮是指：
  执行完当前 执行栈 中的同步任务  
  并清空微任务队列 
  requestAnimationFrame
  执行渲染任务
  然后再执行1个宏任务 为一轮

可以理解为 有几个宏任务 就有几轮 因为执行宏任务前 必须清空微任务队列  
执行完1个宏任务 这一轮就结束
 
●  同步任务 >>> 微任务 >>>  requestAnimationFrame >>> 渲染任务  >>>  1 个宏任务
微任务队列：
  Promise.resolve().then(函数)
  Promise.reject().catch(函数)
  Promise.resolve().finally(函数) 
  queueMicrotask(函数); 
  new window.MutationObserver(函数);
  nextTick(函数);  


requestAnimationFrame(函数)：既不是微任务 也不是宏任务  只是单纯的异步任务 
    这个函数调用完毕 才会执行渲染任务 

渲染任务：既不是微任务 也不是宏任务 只是单纯的异步任务: 
  requestAnimationFrame的回调函数执行完后 宏任务之前 就是渲染任务的完整阶段

宏任务队列：
  setTimeout(函数)
  setInterval(函数) 
  requestIdleCallback(函数)  
  dom.addEventListener('click', 函数) // 所有交互事件都是宏任务 
  // io文件操作也是宏任务

```
