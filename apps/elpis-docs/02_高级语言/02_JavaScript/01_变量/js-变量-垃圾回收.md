# 垃圾回收机制



## 脑图

- 垃圾回收机制
  - 是什么
    - 垃圾回收机制是 JS引擎 自动管理内存的机制，常见垃圾回收算法有 标记清除 引用计数
    - 标记清除是 从根对象出发，递归标记所有可达对象。然后遍历堆内存，回收没有标记的对象。
    - 引用计数是 记录对象的引用次数，大于0就不回收，等于0 就立即回收。
  - 优缺点
    - 标记清除的缺点是 可能产生内存碎片
    - 引用计数的缺点是 无法处理循环引用问题。
  - 讲应用
    -  V8 是把堆内存分为新生代和老生代
      - 新生代使用 scavenge 算法管理生命周期短的变量
      - 老生代使用 标记清除 管理生命周期长的变量。
    - 在平时开发中，不再使用的可达对象 就应该赋值null

## 正文
● 扩展问题
  ○ 内存碎片是什么 ？ 内存碎片就是多个不连续的小块内存
  ○ 标记清除产生的内存碎片如何解决？ 
    ■ 使用标记整理，把标记对象向左侧移动，右侧就是剩余的所有内存
  ○ 循环引用 ？ 两个对象相互引用，解决方法是 使用弱引用或 避免循环引用 
  ○ 内存泄漏 ？内存泄漏就是 不再使用的内存 但是没有回收。
  ○ 什么是垃圾？不能被访问就是垃圾，如何变为垃圾：赋值为 null 保证不能访问。
  ○ 垃圾回收机制是 对 堆内存还是栈内存 回收 ？ 
    ■ 堆内存。原始类型数据用完就销毁 不需要垃圾回收机制介入

// 循环引用举例 
function problem() {
    let objA = {};
    let objB = {};

    objA.ref = objB; // objA 引用 objB
    objB.ref = objA; // objB 引用 objA (循环引用)
}

problem(); // 函数执行完毕，objA 和 objB 已离开作用域，变得不可达
// 对于引用计数来说 他们始终引用次数为1 不能回收 
// 对于标记清除算法来说 从根对象出发 它们两个都不可达 会正常回收
