## 手机浏览器的适配
手机浏览器的适配，由于手机屏幕尺寸繁多，手机浏览器也也不尽相同，

对于手机浏览器的适配，我们 希望 以 375px 或 750px 的宽度作为标准，低于标准就等比缩小，高于标准等比放大。

也就是说 375px 宽度的 手机浏览器上，显示 100 * 100 的盒子，

放到大于 375 的手机上，100 + xxx * 100 + xxx

放到小于 375 的手机上，100 - xxx * 100 - xxx，

padding margin border left fontsize 都是以宽度来缩放



## 方案一 vw 方案


vw 的好处是 不需要依赖 html 的 fontsize 属性，

vw 和 rem 相同点 都是基于视口 document.documentElement.clientWidth

## 设计稿为什么是 375px
因为 iphone6 的屏幕分辨率是 `750 × 1334` , 设备像素比是 2，逻辑分辨率就是 750 / 2 ： 1334 / 2 也就是 `375 × 667`



在 iPhone4 之前 设备像素比都是 1，也就是 逻辑像素和物理像素都是 一一对应的 ，即 css 写宽高为 1px 的盒子 那么物理像素点也就是 1px

iphone4 是划时代产品，实现了 css 的宽高是 1px * 1px，那么物理像素点是 2 * 2 也就是 4 个物理像素点去渲染 1 个 css 像素点更加清晰。



iphone4 `320 × 480` 并没有成为标准 而是 iphone6 成为标准，是因为 iphone6 屏幕相对更大





iphone4之前 所有手机的屏幕 设备像素比 css像素和屏幕分辨率都是 1:1 也就是一一对应的

从2010年的iphone4开始 设备像素比为2 也就是 css像素为320 × 480 而屏幕分辨率为640 × 960 物理像素，这项技术的好处是文字和UI更清晰



2014年的iphone6 开始 设备像素为2 CSS逻辑分辨率：375 × 667，屏幕分辨率：750 × 1334 物理像素，iphone6比iPhone4的屏幕更大 可以显示更多内容



iphone6逐渐成为行业标准，所以到现在为止 都是以iphone6的尺寸作为标准，现在不仅有2倍 还有3倍4倍 甚至更多 但都是以iPhone6为标准的倍数
