## package.json


- package.json

package.json是 包的说明文件，里面描述了包的信息。比如包名 包版本，包依赖。


## 包版本
- 包版本是什么 包版本是package.json的version字段，一般是 3个数字组成，分别是大中小三个版本
大版本，出现重大更新或不兼容才升级
中版本，增加新功能才升级
小版本，是修复bug才升级


## 包依赖
包依赖分为开发依赖和生产依赖 对等依赖
- 开发依赖：package.json中  devDependencies字段 表示，开发时用的依赖，才记录到这里 ，比如 webpack ts eslint 等 用 npm i xx -D添加

- 生产依赖：是package.json中  dependencies字段 表示 比如 vue lodash  用 npm i xx -S添加


- 对等依赖： 是package.json中 peerDependencies字段，当开发框架或组件库时，比如vue组件库，需要vue3.2的版本，用户也有vue版本 必须符合对等依赖记录的版本号。如果开发的组件库基于 vue3.2 一定要写出来 让用户使用vue3.2


- 如何控制依赖版本范围：
包依赖里面 ~表示  大中版本不能变， 小版本可以变。
包依赖里面 ^表示  大版本不能变，中小版本可以改变。 
- ^（脱字符）：表示 中小版本可以变，大版本不能变。
- ~ ：表示 小版本可以变，大中版本不能变。
- 其他符号 https://docs.npmjs.com/cli/v9/configuring-npm/package-json#dependencies

## engines字段 
- engines:用来指定项目需要的 node 版本以及 npm 版本，从而避免用户在使用你的包的时候出现一些因为版本不支持而产生的问题。

```json
"engines": {
  "node": ">=12.0.0",
  "npm": ">=6.0.0"
},
```

## scripts字段
- scripts：配置你的可执行命令

  ```json
  "scripts": {
    "start": "node index.js",
    "test": "jest",
    "build": "webpack",
    "lint": "eslint src",
    "format": "prettier --write src"
  }
  ```

  脚本是可以配置生命周期钩子方法，使用到的关键词为 pre 和 post，pre 代表在执行某个脚本之前，post 代表在执行某个脚本之后

  ```json
  "scripts": {
    "prestart": "npm run build",
    "start": "node index.js",
    "test": "mocha",
    "build": "webpack",
    "lint": "eslint src",
    "format": "prettier --write src",
    "posttest": "npm run lint && npm run format"
  }
  ```

  - prestart：代表在执行 start 之前，先运行 build
  - posttest：代表在执行了 test 之后，同时运行 lint 以及 format
