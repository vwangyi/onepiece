## watch

```vue
<script setup>
import { watch, onWatcherCleanup } from 'vue'

watch(id, (newId) => {
  const { response, cancel } = doAsyncWork(newId)
  // 如果 `id` 变化，则调用 `cancel`，
  // 如果之前的请求未完成，则取消该请求
  onWatcherCleanup(cancel)
})
</script>
<script setup>
const { pause, resume, stop } = watch(source, (newVal, oldVal) => {
  // 监视逻辑
}); 
// 暂停监视
pause(); 
// 恢复监视
resume();
</script>
```

## watchEffect


 

watch 监听到 响应式数据的变化 比如异步获取的数据回来了修改了响应式数据  就可以监听到 然后执行某些方法
再比如 用户修改响应式变量 但程序员不知道用户有没有修改 那就用监听

onBeforeRouteUpdate 有问题 还是用watch好使 
问题1  路由的path更新了 onBeforeRouteUpdate 没有调用
问题2 路由的path更新了 但是拿到的route.query的值还是旧的 



```vue
<script setup>
console.log(1)
watch(
  () => props.xxx,
  // 传入的新值 !== 旧值 时  调用 
  (newValue, oldValue) => {
    // newVal 和 监听的 xxx 是一样的 都是最新值
    console.log(2) 
  },
  {  
    // immediate: true 表示 
    // 会在 初始化阶段执行一次传入的回调函数  且不管传入的新值是否 !== 默认值 
    // 收到的新值为 初始化阶段传入的值 旧值就是undefined 
    // 好处就是 可以拿到初始化阶段传入的值
    // 如果不传immediate: true 
    immediate: true, 
    deep: true, // 深度监听 
  } 
)
```
