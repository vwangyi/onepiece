# 🧅 Koa 洋葱圈模型完全解析

## 一、什么是洋葱圈模型

洋葱圈模型是 Koa 框架最核心的特性，它是一种**中间件执行机制**。顾名思义，就像切开的洋葱一样，一层包着一层，请求和响应会依次穿过这些层级。

### 核心特点
- **先进后出**：中间件的执行顺序像栈一样，先注册的后执行完
- **双向传递**：请求从外到内，响应从内到外
- **完全控制**：可以在请求前后执行任意逻辑

### 可视化理解

```
请求流向：外层 → 内层 → 最内层
响应流向：最内层 → 内层 → 外层

    ╭─────────────────────────────────────╮
    │   Middleware 1 (最外层)              │
    │   ┌─────────────────────────────┐   │
    │   │   Middleware 2              │   │
    │   │   ┌─────────────────────┐   │   │
    │   │   │   Middleware 3      │   │   │
    │   │   │   ┌─────────────┐   │   │   │
    │   │   │   │  业务逻辑    │   │   │   │
    │   │   │   └─────────────┘   │   │   │
    │   │   └─────────────────────┘   │   │
    │   └─────────────────────────────┘   │
    ╰─────────────────────────────────────╯

执行顺序：
1. 进入 Middleware 1 (前半部分)
2. 进入 Middleware 2 (前半部分)
3. 进入 Middleware 3 (前半部分)
4. 执行业务逻辑
5. 退出 Middleware 3 (后半部分)
6. 退出 Middleware 2 (后半部分)
7. 退出 Middleware 1 (后半部分)
```

## 二、基础代码示例

### 1. 最简单的洋葱圈演示

```javascript
const Koa = require('koa');
const app = new Koa();

// 中间件 1
app.use(async (ctx, next) => {
  console.log('1️⃣ 进入中间件1 - 开始');
  await next(); // 调用下一个中间件
  console.log('1️⃣ 退出中间件1 - 结束');
});

// 中间件 2
app.use(async (ctx, next) => {
  console.log('2️⃣ 进入中间件2 - 开始');
  await next();
  console.log('2️⃣ 退出中间件2 - 结束');
});

// 中间件 3
app.use(async (ctx, next) => {
  console.log('3️⃣ 进入中间件3 - 开始');
  await next();
  console.log('3️⃣ 退出中间件3 - 结束');
});

// 业务逻辑
app.use(async (ctx) => {
  console.log('💼 执行业务逻辑');
  ctx.body = 'Hello World';
});

app.listen(3000);

// 访问一次，控制台输出：
// 1️⃣ 进入中间件1 - 开始
// 2️⃣ 进入中间件2 - 开始
// 3️⃣ 进入中间件3 - 开始
// 💼 执行业务逻辑
// 3️⃣ 退出中间件3 - 结束
// 2️⃣ 退出中间件2 - 结束
// 1️⃣ 退出中间件1 - 结束
```

### 2. 带参数的传递

```javascript
  await next();
});

  // 获取所有传递的数据

    user: ctx.state.user,

  };
});
```

## 三、核心原理

### 1. 源码简化版

```javascript
// Koa 中间件执行原理（简化版）
class Koa {
  constructor() {
    this.middleware = []; // 存储所有中间件
  }
  
  // 注册中间件
  use(fn) {
    this.middleware.push(fn);
    return this;
  }
  
  // 组合中间件（核心）
  compose(middleware) {
    return function(context, next) {
      let index = -1;
      
      function dispatch(i) {
        if (i <= index) return Promise.reject('next() called multiple times');
        index = i;
        
        let fn = middleware[i];
        if (i === middleware.length) fn = next;
        if (!fn) return Promise.resolve();
        
        try {
          // 关键：将下一个中间件作为 next 参数传入
          return Promise.resolve(fn(context, dispatch.bind(null, i + 1)));
        } catch (err) {
          return Promise.reject(err);
        }
      }
      
      return dispatch(0);
    };
  }
  
  // 处理请求
  handleRequest(ctx) {
    const fn = this.compose(this.middleware);
    return fn(ctx);
  }
}
```

```
    ↓

```

## 四、实际应用场景

### 1. 日志记录中间件

```javascript
app.use(async (ctx, next) => {
  const start = Date.now();
  
  // 记录请求信息
  console.log(`→ ${ctx.method} ${ctx.url}`);
  
  await next(); // 等待后续处理
  
  // 记录响应信息
  const ms = Date.now() - start;
  console.log(`← ${ctx.method} ${ctx.url} - ${ctx.status} - ${ms}ms`);
});

// 输出示例：
// → GET /api/users
// ← GET /api/users - 200 - 45ms
```

### 2. 错误处理中间件（最重要）

```javascript
app.use(async (ctx, next) => {
  try {
    await next(); // 捕获所有后续中间件的错误
  } catch (err) {
    ctx.status = err.status || 500;
    ctx.body = {
      error: err.message,
      stack: process.env.NODE_ENV === 'development' ? err.stack : undefined
    };
    
    // 记录错误日志
    console.error('Error:', err);
  }
});

// 后面的中间件抛出错误会被捕获
app.use(async () => {
  throw new Error('数据库连接失败');
});
```

### 3. 身份验证中间件

```javascript
app.use(async (ctx, next) => {
  const token = ctx.headers.authorization;
  
  if (!token) {
    ctx.status = 401;
    ctx.body = { error: '未提供认证令牌' };
    return; // 不调用 next()，中断执行
  }
  
  try {
    // 验证token
    const user = verifyToken(token);
    ctx.state.user = user; // 存储用户信息
    await next(); // 验证通过，继续执行
  } catch (err) {
    ctx.status = 403;
    ctx.body = { error: '无效的认证令牌' };
  }
});

// 业务中间件可以直接使用用户信息
app.use(async (ctx) => {
  ctx.body = { message: `Hello ${ctx.state.user.name}` };
});
```

### 4. 性能监控中间件

```javascript
app.use(async (ctx, next) => {
  // 请求开始
  ctx.state.requestId = generateRequestId();
  ctx.state.startTime = process.hrtime();
  
  // 记录请求开始
  console.log(`[${ctx.state.requestId}] Request started`);
  
  await next();
  
  // 计算耗时
  const diff = process.hrtime(ctx.state.startTime);
  const duration = (diff[0] * 1e3 + diff[1] / 1e6).toFixed(2);
  
  // 慢请求告警
  if (duration > 1000) {
    console.warn(`[${ctx.state.requestId}] Slow request: ${duration}ms`);
  }
  
  // 记录响应完成
  console.log(`[${ctx.state.requestId}] Request completed in ${duration}ms`);
});
```

### 5. 请求限流中间件

```javascript
const rateLimit = new Map();

app.use(async (ctx, next) => {
  const ip = ctx.ip;
  const now = Date.now();
  const windowMs = 60 * 1000; // 1分钟窗口
  const maxRequests = 100; // 最多100次请求
  
  if (!rateLimit.has(ip)) {
    rateLimit.set(ip, { count: 1, startTime: now });
    await next();
    return;
  }
  
  const record = rateLimit.get(ip);
  
  if (now - record.startTime > windowMs) {
    // 重置计数
    record.count = 1;
    record.startTime = now;
    await next();
  } else if (record.count < maxRequests) {
    record.count++;
    await next();
  } else {
    // 超过限制
    ctx.status = 429;
    ctx.body = { error: '请求过于频繁，请稍后再试' };
  }
});
```

### 6. 数据库事务中间件

```javascript
app.use(async (ctx, next) => {
  const transaction = await db.startTransaction();
  
  try {
    ctx.state.transaction = transaction;
    await next(); // 执行业务逻辑
    await transaction.commit(); // 提交事务
  } catch (err) {
    await transaction.rollback(); // 回滚事务
    throw err;
  }
});

// 业务中间件使用事务
app.use(async (ctx) => {
  const transaction = ctx.state.transaction;
  await transaction.insert('users', { name: '张三' });
  await transaction.update('accounts', { balance: 100 });
  ctx.body = { success: true };
});
```

## 五、高级用法

### 1. 条件性执行中间件

```javascript
// 只对API请求生效
app.use(async (ctx, next) => {
  if (ctx.path.startsWith('/api')) {
    console.log('API请求:', ctx.path);
    await next();
  } else {
    await next(); // 非API请求直接跳过
  }
});

// 只对特定方法生效
app.use(async (ctx, next) => {
  if (ctx.method === 'POST') {
    console.log('POST请求处理');
    await next();
  } else {
    await next();
  }
});
```

### 2. 中间件组合

```javascript
// 组合多个中间件为一个
function composeMiddleware(middlewares) {
  return async (ctx, next) => {
    let index = -1;
    
    const dispatch = async (i) => {
      if (i <= index) throw new Error('next() called multiple times');
      index = i;
      
      if (i === middlewares.length) {
        return await next();
      }
      
      const middleware = middlewares[i];
      await middleware(ctx, dispatch.bind(null, i + 1));
    };
    
    await dispatch(0);
  };
}

// 使用组合
const authMiddleware = composeMiddleware([
  checkToken,
  checkPermission,
  logAuth
]);

app.use(authMiddleware);
```

### 3. 跳过后续中间件

```javascript
app.use(async (ctx, next) => {
  if (ctx.path === '/health') {
    // 直接返回，不继续执行后续中间件
    ctx.body = { status: 'ok' };
    return; // 不调用 next()
  }
  
  await next(); // 继续执行
});

app.use(async () => {
  // 健康检查路径不会执行到这里
  console.log('正常业务逻辑');
});
```

## 六、常见陷阱和注意事项

### 1. 忘记 await next()

```javascript
// ❌ 错误：没有 await
app.use(async (ctx, next) => {
  console.log('开始');
  next(); // 没有 await，后续代码会先执行
  console.log('结束'); // 这会在 next 执行完之前执行
});

// ✅ 正确
app.use(async (ctx, next) => {
  console.log('开始');
  await next(); // 等待后续中间件执行完
  console.log('结束'); // 确保顺序正确
});
```

### 2. 多次调用 next()

```javascript
// ❌ 错误：多次调用 next()
app.use(async (ctx, next) => {
  await next();
  await next(); // 第二次调用会报错
});

// ✅ 正确：只调用一次
app.use(async (ctx, next) => {
  await next();
  // 后续处理
});
```

### 3. 异步错误处理

```javascript
// ❌ 错误：未捕获的异步错误
app.use(async (ctx, next) => {
  setTimeout(() => {
    throw new Error('异步错误'); // 这个错误不会被捕获
  }, 100);
  await next();
});

// ✅ 正确：使用 Promise
app.use(async (ctx, next) => {
  await new Promise((resolve, reject) => {
    setTimeout(() => {
      reject(new Error('异步错误')); // 可以被捕获
    }, 100);
  });
  await next();
});
```

## 七、与其他框架对比

| 特性 | Koa (洋葱圈) | Express (线性) | 
|------|-------------|---------------|
| 执行模型 | 双向（请求+响应） | 单向（仅请求） |
| 错误处理 | 统一catch | 需要单独处理 |
| 中间件顺序 | 先进后出 | 先进先出 |
| 灵活性 | 非常高 | 一般 |
| 学习曲线 | 中等 | 低 |

## 八、最佳实践总结

1. **错误处理中间件必须放在最前面**
2. **记得使用 `await next()`**
3. **利用 `ctx.state` 在中间件间传递数据**
4. **不要在中间件中执行耗时操作**
5. **合理控制中间件数量（5-10个为宜）**
6. **根据条件决定是否执行 next()**

洋葱圈模型是 Koa 的精髓，理解它就能充分发挥 Koa 的威力！
