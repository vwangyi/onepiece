更完整的说法：事件循环在每次清空微任务后，都会检查“是否需要渲染”，检查条件包括：是否有待处理的视觉更新（DOM/样式变化、动画回调等） 且 是否已达到预期的渲染时机（通常与刷新率对齐）。

如果条件满足，则执行渲染阶段（样式、布局、绘制、合成）；否则跳过，直接进入下一个宏任务。
