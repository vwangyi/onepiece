# 原型链






## 脑图
```js

● 原型和原型链
  ○ 是什么
    ■ 原型就是 每个实例对象都有 __proto__ 每个构造函数都有 prototype 
    ■ 原型链是 实例对象的__proto__ 指向 其构造函数的prototype
  ○ 优缺点
    ■ 原型链的好处是 节约内存。因为所有实例对象的方法都放在其构造函数的prototype 上共享 不管new多少实例都不会浪费内存
    ■ 缺点是访问不存在的成员时 仍然会遍历整个原型链，性能差。
  ○ 应用场景
    ■ 应用场景是 实现继承 扩展类库、共享方法
    ■ 当 访问 对象某个成员时
      ● 先看自身有没有，
      ● 在通过 __proto__ 看 其构造函数的prototype 上有没有
      ● 一直向上找 直到 Object.prototype.__proto__ ===  null 就返回undefined
    ■ 比如，可以在 Vue的原型上 挂载 请求方法，每个页面可以共享这个方法
    ■ Vue.prorortpe.axios = function () {} 



- 当访问对象某个成员时
- 先看自身有没有
- 再通过 __proto__ 看 其构造函数的prototype上有没有
- 一直向上找 直到 Object.prototype.__proto__ === null 就返回undefined


比如 [].toString

先看数组实例自身有没有这个方法 [].toString
再看 Array.prototype.toString 有没有 
再通过 Array.prototype.__prorto__ 看 Object.prototype 有没有
再看 Object.prototype.__proto__ === null 就返回undefined

不管属性和方法 查找逻辑都是这样 属性一般不挂prototype原型上 

（举一个例子在平时开发中可以应用 说明这个八股是有用的）
```
