## 一、Mac 上如何使用 Codex

在 macOS 上，你可以通过以下两种方式使用 Codex：

### 方式一：Codex CLI（终端命令行工具）

这是最基础的使用方式，适合习惯命令行的开发者。

**1. 安装**
打开终端，选择以下任一命令安装：
```bash
# 使用 npm 安装（需要 Node.js 18+）
npm install -g @openai/codex

# 或使用 Homebrew 安装
brew install --cask codex
```
安装完成后，运行 `codex --version` 验证。

**2. 登录认证**
直接运行 `codex` 命令，选择 **Sign in with ChatGPT**，浏览器会打开完成授权。Codex 的使用额度与你的 ChatGPT 套餐（Plus/Pro/Team 等）绑定。

**3. 开始使用**
- **命令模式**：`codex "帮我写一个排序函数"`
- **交互模式（推荐）**：直接输入 `codex` 进入对话界面，可以连续下达指令，AI 会记住上下文。

### 方式二：Codex App（macOS 桌面应用）

OpenAI 还提供了专门的 macOS 桌面应用，目前仅支持 Apple Silicon 芯片的 Mac（M1/M2/M3 及更新机型）。

**1. 下载与安装**
访问 OpenAI Codex 官网下载 macOS 安装包，拖拽安装即可。

**2. 登录与使用**
启动应用后使用 OpenAI 账户登录。你可以通过自然语言描述需求，Codex 会生成代码、创建项目甚至自动化工具。App 界面包含技能（Skills）扩展功能，可以让 Codex 直接操作其他应用的工作区。

## 二、Codex 与 Cursor 的核心区别

简单来说：**Cursor 是你“手边的搭档”，Codex 是你“可以派出去干活的远程员工”。**

| 对比维度 | **Cursor** | **Codex** |
| :--- | :--- | :--- |
| **产品形态** | AI 原生 IDE（基于 VS Code 重构） | 云端代理 + CLI + IDE 扩展 + macOS App |
| **核心理念** | **编辑器优先**：与你实时协作，辅助编码 | **任务优先**：你委派任务，它自主执行 |
| **执行位置** | 本地运行 | 云端沙箱环境（任务隔离，不污染本地） |
| **交互方式** | **同步/实时**：你输入，它立即响应，你看到 diff 再决定 | **异步/并行**：可同时派发多个任务，稍后回来审查结果  |
| **并行能力** | 一次处理一个任务 | **可同时运行多个任务**，互不影响  |
| **视觉反馈** | **优秀**：内联 diff、并排比较、逐块接受/拒绝  | **基础**：主要展示最终结果和执行日志 |
| **模型支持** | 多模型可选（Claude、GPT-5、Gemini 等） | 专注使用 GPT-5.2-Codex（针对编码优化） |
| **定价模式** | $20/月（Pro 固定订阅） | 包含在 ChatGPT Plus（$20/月）或更高套餐中  |
| **开源情况** | 专有软件 | CLI 完全开源（Apache-2.0 协议） |

## 三、怎么选？看你的工作方式

- **选 Cursor，如果你：**
  - 一天大部分时间都在 IDE 里写代码、看 diff、审查改动
  - 需要即时视觉反馈，喜欢逐步指导 AI 完成修改
  - 希望代码保留在本地，重视隐私和离线能力 

- **选 Codex，如果你：**
  - 想把任务“丢出去”让它自己跑，同时去做别的事
  - 需要并行处理多个独立任务（比如同时重构多个模块）
  - 已经订阅了 ChatGPT Plus/Pro，希望“零成本”获得编码代理能力 

---

## 四、一个务实的选择

对于大多数开发者来说，**双持**可能是最高效的策略：用 **Cursor** 作为日常主力编辑器享受流畅的协作体验，同时在需要并行任务或风险操作时调用 **Codex** 作为辅助工具。这两者不是替代关系，而是互补的。
