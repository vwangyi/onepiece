# 包入口
- https://docs.npmjs.com/cli/v11/configuring-npm/package-json#browser

- https://docs.npmjs.com/cli/v11/configuring-npm/package-json#main

```json
{
  "main": "./main.js", // node环境的 包的入口文件 默认值为 index.js
  "browser": "browser.js", // 浏览器环境的 包的入口文件

  // 应用场景：有时候我们的库支持 两种环境 浏览器环境和node环境 只有入口文件不同
  "browser": {
     "./node-version.js": "./browser-version.js", // 浏览器环境下 用browser-version 替换node-version
  },
  "browser": {
     "fs": false, // 排除fs模块  浏览器环境下没有fs模块
  },
  "module": "", // 为打包工具（如 Webpack, Rollup）提供一个 ES Module (ESM) 格式的入口。

  // 当 exports 字段存在时，它的优先级最高，会完全覆盖 main、module、browser 等旧字段，所有模块解析逻辑都以它为准。
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  },
  // https://nodejs.org/learn/modules/publishing-a-package#attach-named-exports-directly-onto-exports
  "exports": {
    "types": "", // 告诉 TypeScript 去哪里找类型定义（.d.ts）
    "import": "./dist/index.esm.js", // 当使用者通过 import 导入你的包时匹配
    "require": "./dist/index.cjs",  // 当使用者通过 require() 导入你的包时匹配
    "node": "",
    "default": "", // 当前面所有条件都不匹配时，使用此处的配置，建议始终保留
  }
}
```

- rowser字段：指定浏览器环境的入口文件
- import 
- exports

## 包执行相关配置

- *main*：代表包的入口文件

- browser：该选项表示如果是在浏览器环境下，可以替换一些特定模块或者文件。

  - 指定浏览器的入口文件

  ```json
  {
    "main": "index.js",
    "browser": "browser.js"
  }
  ```

  上面的配置表面在 node 环境下，index.js 为入口文件，如果是浏览器环境，使用 browser.js 作为入口文件

  - 替换特定的模块：浏览器环境时 用browser-version文件替换 node-version

  ```json
  {
    "browser": {
      "./node-version.js": "./browser-version.js"
    }
  }
  ```

  - 排除某些模块：浏览器环境打包时，fs模块排除在外不用打包 因为浏览器环境不能用fs

  ```json
  {
    "browser": {
      "fs": false
    }
  }
  ```



node 常见的配置项除了 type 以外，还有一个 exports，该配置项用于定义一个模块的导出映射，通过这个配置项，可以对模块的导入环境以及条件做一个更精细的控制，指定不同的模块的入口文件。

```
{
  "exports": {
    "import": "./dist/index.esm.js",
    "require": "./dist/index.cjs"
  }
}
```

我们在这里为不同的模块系统提供了不同的入口文件

- 使用的 ESM，那么在导入模块的时候，node.js 会去加载 index.esm.js
- 使用的是 commonjs，那么在导入模块的时候，node.js 会去加载 index.cjs
