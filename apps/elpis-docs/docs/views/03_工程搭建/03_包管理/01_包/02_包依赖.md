# 包依赖 


## 开发依赖
- devDependencies
devDependencies 表示开发依赖 npm i xxx -D // -D 等价 --save-dev 

- 开发依赖：package.json中  devDependencies字段 表示，开发时用的依赖，才记录到这里 ，比如 webpack ts eslint 等 用 npm i xx -D添加

## 生产依赖 
- dependencies
dependencies 表示生产依赖 npm i xxx -S // -S 等价 --save   是默认的 可以不写 
- 生产依赖：是package.json中  dependencies字段 表示 比如 vue lodash  用 npm i xx -S添加


## 对等依赖  
- peerDependencies
- 对等依赖： 是package.json中 peerDependencies字段，当开发框架或组件库时，比如vue组件库，需要vue3.2的版本，用户也有vue版本 必须符合对等依赖记录的版本号。如果开发的组件库基于 vue3.2 一定要写出来 让用户使用vue3.2

## 包依赖版本范围 

"^3.2.0" 表示：^（插入符）表示允许 次版本和补丁版本 自动更新，不允许主版本自动更新 

"~3.2.0" 表示：~（波浪号）表示 补丁版本自定更新 主版本和次版本都不会更新。

"3.2.0" 表示：无符号表示精确版本，不自动更新。 
- 如何控制依赖版本范围：
包依赖里面 ~表示  大中版本不能变， 小版本可以变。
包依赖里面 ^表示  大版本不能变，中小版本可以改变。 
- ^（脱字符）：表示 中小版本可以变，大版本不能变。
- ~ ：表示 小版本可以变，大中版本不能变。
- 其他符号 https://docs.npmjs.com/cli/v9/configuring-npm/package-json#dependencies 


## 开发依赖和生产依赖区别

- 如果构建业务项目 前端的SPA项目 基本没有什么区别 
- 如果构建基建项目 组件库 工具库 函数库 就有区别

- 生产依赖和开发依赖区别 devDependencies 和 dependencies
