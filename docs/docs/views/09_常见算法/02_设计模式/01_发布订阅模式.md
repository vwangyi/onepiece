# 发布订阅模式


## 发布订阅模式（Publish-Subscribe Pattern）

发布订阅是一种**消息通信模式**，它定义了对象之间的一对多依赖关系：当一个对象（发布者）的状态发生变化时，所有依赖它的对象（订阅者）都会自动收到通知。

## 核心概念

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────┐
│   Publisher │────▶│   Event Channel  │────▶│ Subscriber  │
│  (发布者)    │     │    (事件中心)     │     │  (订阅者)    │
└─────────────┘     └──────────────────┘     └─────────────┘
                            │
                            ▼
                    ┌─────────────┐
                    │ Subscriber  │
                    │  (订阅者)    │
                    └─────────────┘
```

### 关键角色

| 角色 | 职责 | 例子 |
|------|------|------|
| **发布者** | 发布事件/消息，不关心谁接收 | 用户点击按钮 |
| **订阅者** | 订阅感兴趣的事件，收到通知后执行操作 | 点击后更新UI |
| **事件中心** | 维护订阅关系，负责消息分发 | Event Bus |

## 简单实现

### JavaScript 版本

```javascript
class EventBus {
    constructor() {
        this.events = {};  // 存储订阅关系
    }
    
    // 订阅事件
    on(eventName, callback) {
        if (!this.events[eventName]) {
            this.events[eventName] = [];
        }
        this.events[eventName].push(callback);
        
        // 返回取消订阅函数
        return () => this.off(eventName, callback);
    }
    
    // 发布事件
    emit(eventName, ...args) {
        const callbacks = this.events[eventName];
        if (callbacks) {
            callbacks.forEach(cb => cb(...args));
        }
    }
    
    // 取消订阅
    off(eventName, callback) {
        const callbacks = this.events[eventName];
        if (callbacks) {
            this.events[eventName] = callbacks.filter(cb => cb !== callback);
        }
    }
    
    // 只订阅一次
    once(eventName, callback) {
        const wrapper = (...args) => {
            callback(...args);
            this.off(eventName, wrapper);
        };
        this.on(eventName, wrapper);
    }
}

// 使用示例
const bus = new EventBus();

// 订阅者 A
const unsubscribeA = bus.on('userLogin', (user) => {
    console.log(`A 收到: ${user.name} 登录了`);
});

// 订阅者 B
bus.on('userLogin', (user) => {
    console.log(`B 收到: ${user.name} 登录了，更新欢迎语`);
});

// 发布者
bus.emit('userLogin', { name: '张三', id: 123 });
// 输出:
// A 收到: 张三 登录了
// B 收到: 张三 登录了，更新欢迎语

// 取消订阅
unsubscribeA();

bus.emit('userLogin', { name: '李四', id: 456 });
// 输出:
// B 收到: 李四 登录了，更新欢迎语
```

### TypeScript 版本（带类型安全）

```typescript
// 定义事件类型映射
type EventMap = {
    userLogin: { name: string; id: number };
    userLogout: { name: string };
    dataUpdate: { key: string; value: any };
    error: { message: string; code: number };
};

class TypedEventBus<T extends Record<string, any>> {
    private events: Partial<Record<keyof T, Array<(...args: any[]) => void>>> = {};
    
    // 订阅事件（带类型约束）
    on<K extends keyof T>(eventName: K, callback: (data: T[K]) => void) {
        if (!this.events[eventName]) {
            this.events[eventName] = [];
        }
        this.events[eventName]!.push(callback);
        
        return () => this.off(eventName, callback);
    }
    
    // 发布事件
    emit<K extends keyof T>(eventName: K, data: T[K]) {
        const callbacks = this.events[eventName];
        if (callbacks) {
            callbacks.forEach(cb => cb(data));
        }
    }
    
    // 取消订阅
    off<K extends keyof T>(eventName: K, callback: (data: T[K]) => void) {
        const callbacks = this.events[eventName];
        if (callbacks) {
            this.events[eventName] = callbacks.filter(cb => cb !== callback);
        }
    }
    
    // 只订阅一次
    once<K extends keyof T>(eventName: K, callback: (data: T[K]) => void) {
        const wrapper = (data: T[K]) => {
            callback(data);
            this.off(eventName, wrapper);
        };
        this.on(eventName, wrapper);
    }
}

// 使用示例
const bus = new TypedEventBus<EventMap>();

// ✅ 类型安全，自动提示参数类型
bus.on('userLogin', (user) => {
    console.log(user.name);  // TypeScript 知道 user 有 name 属性
});

// ❌ 编译错误：参数类型不匹配
// bus.emit('userLogin', { name: '张三' });  // 缺少 id 属性

// ✅ 正确
bus.emit('userLogin', { name: '张三', id: 123 });
```

## 实际应用场景

### 1. Vue.js 中的事件总线

```javascript
// Vue 2 中常见用法
const EventBus = new Vue();

// 组件 A：发布事件
EventBus.$emit('data-updated', { id: 1 });

// 组件 B：订阅事件
EventBus.$on('data-updated', (data) => {
    console.log('数据更新:', data);
});

// 组件销毁前取消订阅
beforeDestroy() {
    EventBus.$off('data-updated');
}
```

### 2. React 组件通信

```javascript
// 使用 mitt（轻量级事件库）
import mitt from 'mitt';

const emitter = mitt();

// 组件 A：发布
emitter.emit('cartUpdate', { count: 5 });

// 组件 B：订阅
useEffect(() => {
    const handler = (data) => console.log(data);
    emitter.on('cartUpdate', handler);
    return () => emitter.off('cartUpdate', handler);
}, []);
```

### 3. 跨组件通信

```javascript
// 购物车场景
class CartEventBus {
    constructor() {
        this.bus = new EventBus();
    }
    
    addItem(item) {
        // 添加商品到购物车
        this.cart.push(item);
        // 发布事件，通知所有订阅者
        this.bus.emit('cart:itemAdded', item);
        this.bus.emit('cart:updated', this.cart);
    }
    
    removeItem(id) {
        this.cart = this.cart.filter(item => item.id !== id);
        this.bus.emit('cart:itemRemoved', id);
        this.bus.emit('cart:updated', this.cart);
    }
}

// 不同组件订阅不同事件
// 购物车图标组件
cartBus.on('cart:updated', (cart) => {
    updateBadge(cart.length);
});

// 商品列表组件
cartBus.on('cart:itemAdded', (item) => {
    showToast(`已添加 ${item.name} 到购物车`);
});

// 结算组件
cartBus.on('cart:updated', (cart) => {
    updateTotalPrice(calculateTotal(cart));
});
```

## 发布订阅 vs 观察者模式

| 对比维度 | 发布订阅模式 | 观察者模式 |
|----------|------------|----------|
| **耦合度** | 低（通过事件中心解耦） | 中（观察者需要知道被观察者） |
| **通信方式** | 间接通信（通过事件中心） | 直接通信（观察者直接注册到被观察者） |
| **灵活性** | 高（可以跨模块通信） | 中（通常在同一对象内） |
| **使用场景** | 跨组件、跨模块通信 | 内部状态变化通知 |
| **典型实现** | EventBus、消息队列 | Vue 响应式系统、RxJS |

## 优缺点

### ✅ 优点
- **解耦**：发布者和订阅者完全解耦，互不依赖
- **扩展性**：轻松添加新的订阅者，无需修改发布者
- **灵活性**：支持一对多通信，可以动态订阅/取消
- **异步支持**：可以实现异步消息处理

### ❌ 缺点
- **调试困难**：事件流向不明确，难以追踪
- **内存泄漏**：忘记取消订阅可能导致内存泄漏
- **顺序问题**：订阅者执行顺序不确定
- **过度使用**：滥用会导致代码难以维护

## 最佳实践

```javascript
// 1. 统一管理事件名称（避免拼写错误）
const Events = {
    USER_LOGIN: 'user:login',
    USER_LOGOUT: 'user:logout',
    DATA_UPDATED: 'data:updated',
    ERROR_OCCURRED: 'error:occurred'
};

// 2. 组件卸载时取消订阅
class MyComponent {
    constructor() {
        this.unsubscribe = bus.on(Events.USER_LOGIN, this.handleLogin);
    }
    
    destroy() {
        this.unsubscribe(); // 清理订阅
    }
}

// 3. 使用命名空间避免冲突
const bus = new EventBus();
const userBus = new EventBus();  // 用户相关事件
const orderBus = new EventBus(); // 订单相关事件

// 4. 错误处理
bus.on('data:fetch', async () => {
    try {
        await fetchData();
    } catch (error) {
        bus.emit('error:fetch', error);
    }
});
```

发布订阅模式是现代前端开发中非常重要的设计模式，它让组件间的通信更加灵活和解耦，但需要谨慎使用，避免过度依赖导致代码难以维护。
