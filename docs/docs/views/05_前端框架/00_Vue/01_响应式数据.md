## 概念



数据响应式实现

是什么：数据响应式是 数据改变后，依赖该数据的函数重新执行。常见的方案有 defineProperty 和 Proxy 。

优缺点：defineProperty缺点  针对 单个属性的监听
       Proxy 的好处proxy是针对整个对象监听，可以解决defineProperty的所有缺点

讲应用：平时开发中 需要渲染的数据定义为 响应式数据，不需要渲染的数据定义为普通变量



初级回答 优缺点 深度回答 观察者模式和 发布订阅模式

观察者模式和 发布订阅模式的区别



defineProperty 的缺点：
1 只是针对单个属性监听 
2 后续追加的属性 不支持监听
3 影响原数组方法修改 也不支持监听

getter：当读取数据时，记录是哪些函数读取数据
setter：当修改数据时，就调用之前记录的函数。

### 追问
- Watcher 和 Dep 是多对多吗？	是的。一个 Watcher 对应多个 Dep（一个组件用到了多个响应式属性），一个 Dep 也对应多个 Watcher（一个属性被多个组件使用）。
- Vue 2 为什么不能检测数组索引修改？	defineProperty 可以监听索引，但出于性能考虑 Vue 没做。因为数组可能很大，监听每个索引成本高，所以只 hack 了 7 个变异方法。
- Vue 3 的 Proxy 有什么缺点？	无法兼容 IE，且不能直接代理原始值（如 ref 就是为了解决这个问题）。
- nextTick 和响应式有什么关系？	Vue 的更新是异步批量的。数据变化后不会立即更新 DOM，而是把 Watcher 放入队列，nextTick 的回调会在队列清空、DOM 更新后执行。

## props + toRef
```js
const toRefData = toRef(props, 'data')
const toRefFocused = toRef(props, 'focused')
const showBorder11 = toRef(props, 'showBorder1')
 


const {showsss} = usexixi(toRefData, toRefFocused, showBorder11); // showsss 是响应式的
```



这是一个非常经典且重要的问题。它们的核心区别在于**是否与源数据保持“连接”**。

针对你的场景（`props` 是 Vue 的响应式对象），区别如下：

### 1. `toRef(props, 'data')` —— **保持连接（推荐）**
- **行为**：创建一个 **引用（Ref）**，但它**指向的是 `props.data` 这个属性本身**。
- **响应式**：当你修改 `props.data` 的值时，`toRef` 返回的 ref 的 `.value` 会同步变化；反之，修改这个 ref 的 `.value`，也会同步修改 `props.data`（虽然你不应该直接修改 props）。
- **本质**：它只是一个**指针**（或者说“视图”），没有复制数据。
- **适用场景**：**必须用于响应式对象（如 `props`、`reactive` 对象）的属性**。

```javascript
const props = reactive({ data: { id: 1 }, focused: false });

const dataRef = toRef(props, 'data');

// ✅ props.data 变化时，dataRef.value 自动更新
props.data = { id: 2 };
console.log(dataRef.value); // { id: 2 }

// ✅ 修改 dataRef 也会影响 props（但通常不推荐）
dataRef.value = { id: 3 };
console.log(props.data); // { id: 3 }
```

---

### 2. `ref(props.data)` —— **断开连接（复制值）**
- **行为**：创建一个**全新的独立的 Ref**，并将 `props.data` 的**当前值**复制进去。
- **响应式**：它只保存了创建那一刻的**值（快照）**。之后 `props.data` 如何变化，都与这个 `ref` **无关**。
- **本质**：它是一个**副本**。
- **适用场景**：当你想保存一个**历史快照**，或者数据不是响应式对象时。

```javascript
const props = reactive({ data: { id: 1 }, focused: false });

const dataRef = ref(props.data); // 此时 dataRef.value === { id: 1 }

// ❌ props.data 变化时，dataRef.value 不会更新
props.data = { id: 2 };
console.log(dataRef.value); // { id: 1 }  👈 还是旧值

// ✅ 修改 dataRef 也不会影响 props
dataRef.value = { id: 3 };
console.log(props.data); // { id: 2 }  👈 没有变化
```

## 针对你的代码场景

在你的 `useGeneralItem` 中：

```javascript
// ❌ 错误做法：断开了连接
const { showBorder } = useGeneralItem(ref(props.data), ref(props.focused));
// 当 props.focused 变化时，showBorder 不会更新
```

```javascript
// ✅ 正确做法：保持连接
import { toRef } from 'vue';

const { showBorder } = useGeneralItem(
    toRef(props, 'data'),      // 保持与 props.data 的连接
    toRef(props, 'focused')    // 保持与 props.focused 的连接
);
```

## 深层对比表格

| 特性 | `toRef(props, 'data')` | `ref(props.data)` |
|------|------------------------|-------------------|
| **数据来源** | 指向原属性的引用 | 复制当前值（快照） |
| **原属性变化时** | ✅ 同步更新 | ❌ 不会更新 |
| **修改 ref 时** | ✅ 同步修改原属性 | ❌ 只修改副本 |
| **适用于** | `reactive` 对象的属性 | 普通值、或需要快照的场景 |
| **性能** | 几乎无开销（只是引用） | 需要复制数据（可能深拷贝） |
| **类型安全** | 需要指定属性名（字符串） | 直接传值 |

## 特殊情况：`toRefs` 批量转换

如果你想把 `props` 的**所有属性**都转为 ref，可以用 `toRefs`：

```javascript
import { toRefs } from 'vue';

const props = defineProps({
    data: Object,
    focused: Boolean,
});

// 一次性将所有 props 转为响应式 ref
const { data, focused } = toRefs(props);

// 现在 data 和 focused 都保持与 props 的连接
const { showBorder } = useGeneralItem(data, focused);
```

## 为什么你的 `showBorder` 不响应？

回顾你的代码：

```javascript
const props = defineProps({
    data: Object,
    focused: { type: Boolean, default: false },
});

// ❌ 问题在这里
const { showBorder } = useGeneralItem(props.data, props.focused);
```

当 `props.focused` 变化时：
- 你传给 `useGeneralItem` 的是**普通值**（`false` 或 `true`）
- 函数内部的 `focused` 参数只是一个**普通变量**，不会感知到变化
- 所以 `showBorder` 的 computed 依赖的是旧值

**修复**：

```javascript
import { toRef } from 'vue';

const { showBorder } = useGeneralItem(
    toRef(props, 'data'),
    toRef(props, 'focused')  // 👈 保持连接
);
```

## 在 `useGeneralItem` 内部的适配

为了让 `useGeneralItem` 兼容两种传参方式，可以这样写：

```javascript
import { isRef, unref, computed } from 'vue';

export default function useGeneralItem(dataSource, focusedSource) {
    // 如果传入的是 ref，直接使用；如果是普通值，转为 ref
    const dataRef = isRef(dataSource) ? dataSource : ref(dataSource);
    const focusedRef = isRef(focusedSource) ? focusedSource : ref(focusedSource);
    
    // 所有依赖都用 computed 访问 .value
    const data = computed(() => dataRef.value);
    const focused = computed(() => focusedRef.value);
    
    const props = computed(() => data.value?.props ?? {});
    
    const showBorder = computed(() => 
        (props.value?.showBorder ?? false) && focused.value
    );
    
    // ...
}
```

## 一句话总结

- **`toRef(props, 'data')`**：**保持连接**，适合传递 props 属性
- **`ref(props.data)`**：**复制快照**，适合保存历史值或非响应式数据

在你的场景中，**必须使用 `toRef`**，才能让 `showBorder` 响应 `props.focused` 的变化。
