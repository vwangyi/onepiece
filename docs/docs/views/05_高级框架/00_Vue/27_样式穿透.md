# 样式穿透


在 Vue 中使用**样式穿透**（也叫深度作用选择器），主要是为了在 `scoped` 样式中能够修改**子组件内部**或**第三方组件**的样式。由于 `scoped` 会给当前组件的元素添加一个唯一属性（如 `data-v-xxx`），导致普通的选择器无法作用到子组件的根节点以外的元素。


## `:deep()`  `vue3推荐`

```vue 
<template>
  <a-input class="my-input" placeholder="测试" />
</template>

<style scoped lang="scss">  
.my-input :deep(.ant-input) { 
  color: red;
} 

.parent :deep(.grand .target) {} /* 穿透多层*/
.parent :deep(.target:hover) {} /* 伪类结合 */

.my-input ::v-deep .ant-input { }
.my-input /deep/ .ant-input { }
.my-input >>> .ant-input { }

</style>
``` 
## 注意事项

1. **只在需要穿透时使用**：滥用样式穿透会破坏组件的封装性，增加维护成本。优先通过组件的 props / slots 定制样式。
2. **优先级问题**：穿透后的样式依然遵循 CSS 优先级规则，必要时可以增加父级选择器权重或使用 `!important`。
3. **不要在全局样式（不带 scoped）中用穿透**：全局样式本来就影响所有子组件，无需穿透。
4. **Vue 3 中 `::v-deep` 已标记为废弃**：官方推荐使用 `:deep()`，但 `::v-deep` 在 3.x 中仍然可用（会有警告）。

### 快速对比表

| 需求 | 写法 (Vue 2) | 写法 (Vue 3) |
| :--- | :--- | :--- |
| 穿透一层 | `.parent ::v-deep .target` | `.parent :deep(.target)` |
| 穿透多层（嵌套） | `.parent ::v-deep .grand .target` | `.parent :deep(.grand .target)` |
| 与伪类结合 | `.parent ::v-deep .target:hover` | `.parent :deep(.target:hover)` |

### 常见的样式穿透写法

| 场景 / 版本 | 写法 | 示例 |
| :--- | :--- | :--- |
| **Vue 2** (推荐) | `::v-deep` | `.parent ::v-deep .child-class { color: red; }` |
| Vue 2 (旧式) | `/deep/` 或 `>>>` | `.parent /deep/ .child-class { }` |
| **Vue 3** (推荐) | `:deep()` | `.parent :deep(.child-class) { color: red; }` |
| Vue 3 (别名) | `::v-deep` 仍可用，但建议用 `:deep()` | – |

> 注意：`>>>` 和 `/deep/` 在某些预处理器（如 SCSS）中可能解析失败，推荐统一使用 `::v-deep` (Vue 2) 或 `:deep()` (Vue 3)。
 
## 总结 

1. 优先通过组件提供的api进行修改
2. 不行才用样式穿透
3. 终极方案 操作原生DOM 解决样式问题
