## 代码质量 eslint
- eslint官网  https://zh-hans.eslint.org/
- 安装eslint >>> npm i eslint 
- 书写配置项 >>> eslint.config.js
- 忽略 >>> .eslintignore
- 命令式检查 >>> npm run eslint  
- vscode插件检查 >>> eslint 

## 代码风格 prettier 
- prettier官网 
- 安装 prettier >>> npm i prettier 
- 书写配置项 >>> prettier.config.js
- 忽略 >>> .prettierignore
- 命令式检查 >>> npm run prettier  
- 命令式修复 >>> pnpm eslint --fix apps/elpis-koa/main.js
- vscode插件检查和修复 >>> prettier  


## prettier和eslint
- 存在交集 比如 是否使用分号码 既可以说是格式也可以说是质量 取决于自己认为是什么 就用对应的工具解决 
- eslint 主要是检查 
- perttier 主要是修复

- 比如 
- 分号 空格 属于代码风格
- var const 属于代码质量

## 拼写检查

- 安装环境 >>> pnpm -Dw add cspell @cspell/dict-lorem-ipsum
- 配置 >>> touch cspell.json
- 自定义字典 >>> mkdir -p ./.cspell && touch ./.cspell/custom-dictionary.txt
- 命令式检查 >>> "lint:spellcheck"
- vscode插件修复： Code Spell Checker

## 提交规范

- lint-staged
- lint-staged的作用是  git add 添加暂存区时 检查 主要是拦截git add 
- 安装lint-staged >>> pnpm -Dw add lint-staged
- 配置命令 >>>  "precommit": "lint-staged"
- 配置文件 >>> .lintstagedrc.js



- husky
- 安装husky pnpm -Dw add husky  
- 初始化生成 .husky目录  >>> pnpx husky init  >>> 
- husky的作用是。连接git hooks 可以再提交之前之后做做检查，husky主要是拦截 git commit 
- 提交之前做些事
```cmd
#!/usr/bin/env sh
pnpm lint:prettier && pnpm lint:eslint && pnpm lint:spellcheck
```

- commitlint
- commitlint的作用是 提供交互式命令 做提交 来替换 git commit 原生命令
- 安装环境 pnpm -Dw add @commitlint/cli @commitlint/config-conventional commitizen cz-git
- 书写配置文件 >>> touch commitlint.config.js 
- 命令式提交
```json 
// package.json
"scripts": {  
	"commit": "git-cz"
},
// 额外有一个配置在这
"config": {
  "commitizen": {
    "path": "node_modules/cz-git"
  }
} 
```
