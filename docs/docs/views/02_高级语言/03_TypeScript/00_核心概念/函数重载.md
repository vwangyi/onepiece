在 TypeScript 中，函数重载允许你为同一个函数定义多个不同的调用方式。它通过**声明多个重载签名**（调用时的类型约束）和一个**实现签名**（实际的函数体）来实现。

## 基本语法

```typescript
// 1. 重载签名（多个）
function greet(name: string): string;
function greet(age: number): string;

// 2. 实现签名（一个）
function greet(value: string | number): string {
  if (typeof value === "string") {
    return `Hello, ${value}`;
  } else {
    return `Age: ${value}`;
  }
}

// 使用
greet("Alice"); // 正确
greet(25);      // 正确
// greet(true); // 错误：参数类型不匹配
```

## 不同参数数量的重载

```typescript
function sum(a: number, b: number): number;
function sum(a: number, b: number, c: number): number;
function sum(a: number, b: number, c?: number): number {
  return c ? a + b + c : a + b;
}
```

## 返回值类型不同的重载

```typescript
function getData(id: number): { id: number; name: string };
function getData(name: string): { id: number; name: string }[];
function getData(value: number | string): any {
  if (typeof value === "number") {
    return { id: value, name: "Item" };
  } else {
    return [{ id: 1, name: value }];
  }
}
```

## 类中的方法重载

```typescript
class Calculator {
  add(a: number, b: number): number;
  add(a: string, b: string): string;
  add(a: any, b: any): any {
    if (typeof a === "number" && typeof b === "number") {
      return a + b;
    } else if (typeof a === "string" && typeof b === "string") {
      return a.concat(b);
    }
  }
}
```

## 重要注意事项

1. **实现签名必须兼容所有重载签名**：通常使用联合类型、可选参数或 `any` 类型。
2. **重载签名不能有实现体**：只有最后一个实现签名包含函数体。
3. **TypeScript 的重载是编译时的**：编译后的 JavaScript 只是一个普通函数（没有重载痕迹）。
4. **尽量优先使用联合类型**：如果可以通过联合类型实现，比重载更简洁。例如上面的 `greet` 其实可以写成：
   ```typescript
   function greet(value: string | number): string { ... }
   ```
   只有当联合类型无法精确表达（比如参数数量或类型组合导致返回值类型不同）时，才需要重载。

## 箭头函数不支持重载

```typescript
// ❌ 错误
const greet = (name: string): string;
const greet = (age: number): string;

// ✅ 正确：改用函数声明或内部判断
const greet = (value: string | number): string => {
  // ...
};
```

掌握重载可以帮助你为复杂参数模式提供清晰的类型提示，但不要过度使用。能用联合类型或可选参数解决的，优先选择更简单的方式。
