## 并发和并行

并发是 在一段时间内 CPU面对多个任务，会讲每个任务交替执行一段时间。对于某个瞬间只在执行一个任务，高频切换 让每个任务都能得到推进。并不是同时执行的。

并行是多个CPU或多核的单个CPU，在同一时间，每个核心都在执行不同的任务。多个任务在同时执行。

获取 几核 os模块  
并发是逻辑上的“同时”，是任务交替执行（看起来像同时）。
并行是物理上的“同时”，是任务真正在同时执行。

单核CPU通过快速切换任务执行，实现同时处理多个任务。
多核CPU在同一时间执行不同的任务，实现真正的同时处理。

在现在的操作系统中 并发和并行是同时存在的。 
从单核CPU看 那就是并发 
从多核CPU看 那就是并行

举个例子。并发是一个人吃三个馒头，一口一口交替着吃。并行是三个人一人吃一个馒头，同时吃。

## 进程和线程

process 线程thread
是什么 

- `进程是操作系统 分配资源的基本单位。也就是说 分配资源是以进程为单位。`
- `创建一个进程 就等同于 创建一个独立的内存空间，内存空间中至少有一个线程。这个初始线程一般叫主线程。`

- `线程是操作系统调用CPU的基本单元。也就是说 调用CPU执行代码是以线程为单位。线程是真正执行代码的人。进程内的所有线程都共享同一块内存。`

- 进程之间 内存是隔离的，
- 线程之间, 都是共享进程的内存。

- 进程的好处是 内存是隔离的 但可以相互通信，一个进程崩溃 不影响其他进程。
- 线程的好处是 同一个进程内的所有线程 都共享同一块内存。


- 打开浏览器会创建浏览器进程，打开标签页会创建渲染进程。

## 执行栈（执行上下文栈） 
- 执行栈是 使用先进后出的栈结构 来管理 执行上下文。 一开始栈底是全局执行上下文，每调用一个函数都会创建对应的函数执行上下文 压入栈底。如果递归没有结束条件，最终就会导致栈溢出。
 
- 为什么需要执行栈？  js是单线程语言 一次只能做一件事，执行栈可以
1. 管理函数的执行顺序 
2. 管理多个执行上下文的创建和销毁 
3. 记录当前执行到了哪里 
4. 处理函数的返回 


执行栈和异步代码的关系 

- 执行栈里面是当前执行的函数
- 任务队列里面是等待执行的函数， 异步任务不会立即执行，而是通过事件循环进行排队。



## 执行上下文 
- 全局执行上下文 函数执行上下文 每次创建都会压入执行栈中

- 每次调用函数（不是定义）时，就创建 函数执行上下文
- 执行上下文：分为 全局执行上下文 函数执行上下文 块级执行上下文 （eval()执行上下文 esm执行上下文 箭头函数没有自己的执行上下文，它使用父级的：）

- 执行上下文就是用来记录 函数的 词法环境 this 作用域链 等等 变量对象 等等  一般有三种 全局函数 eval 执行上下文

## 编译阶段 和 执行阶段

- 编译阶段
    1. 词法分析（Lexing）：代码 → Token 流
    2. 语法分析（Parsing）：Token → AST（抽象语法树）
    3. 作用域确定：确定每个标识符在哪个作用域
    4. 生成字节码或机器码
    5. 【关键】创建全局执行上下文（结构确定，部分初始化）
- 执行阶段
    1. 逐条执行代码
    2. 遇到 var x = 5 → 赋值
    3. 遇到函数调用 → 创建新的执行上下文
    4. 逐行执行直到结束

## 变量提升和暂时性死区
- var变量 可以在声明前访问 是 在执行上下文里面 已经存储了 初始值为undefined的var变量 
- let/const 变量 存在暂时性死区是 ？ 


- var变量可以在声明前访问，是因为执行上下文里面记录了 var变量 并且初始化值为 undefined 并可以访问 没有死区。
- letconst变量 不能在声明前访问，，在执行上下文里面 也记录了let变量 但没有初始化为undefined，并且在赋值前都属于死区。访问就报错。

这样设计的好处是 避免 var 带来的提升混乱，强制变量 遵循 先声明 后使用的规范。


## 堆和栈 

- 当打开浏览器时，会分配浏览器进程，浏览器进程里面 有一块内存空间和至少一个线程。
- 内存空间就分为 堆内存和栈内存
- 栈内存 存储 基本数据类型和 指向堆内存的引用数据类型的地址（只是地址）
    - 函数调用时创建执行上下文（Execution Context）
    - LIFO（后进先出）结构
    - 函数执行完通常会被销毁 不需要垃圾回收机制介入 
- 堆内存 存储    
    - 存储引用类型的实际数据（对象、数组、函数体等）
    - 需要手动或自动（垃圾回收）管理 需要垃圾回收机制管理

    注意：函数内部的基本数据类型，虽然是基本数据类型 但因为它是函数内的局部变量且有嵌套函数引用了它，所以不在栈上，而是分配在堆中。

## 词法环境 

- 每个函数执行时，会创建自己的执行上下文，包含三部分：

```js
ExecutionContext = {
  ThisBinding: this值,
  LexicalEnvironment: {  // 词法环境
    EnvironmentRecord: {},  // 存储变量和函数声明
    outer: reference  // 指向外部词法环境（作用域链核心）
  },
  VariableEnvironment: {}
}
```
- 
```js
function outer() {
  let count = 0;        // ① 在堆中分配
  
  function inner() {    // ② inner函数体也在堆中
    count++;            // ③ 引用外部变量
    console.log(count);
  }
  
  return inner;         // ④ 返回inner函数的引用
}

const fn = outer();     // ⑤ 外部变量fn持有inner引用
fn();                   // ⑥ 执行inner
```
关键点：count 虽然是基本类型，但因为它是函数内的局部变量且有嵌套函数引用了它，所以不在栈上，而是分配在堆中。
