 
## 事件循环
- 事件循环是 为了解决单线程语言 如何调度任务 实现异步编程的核心机制, 简单来说就是排队。
- 排队规则是：先同步后异步 先微任务后宏任务，执行栈每次执行宏任务前 都会轮询微任务队列 清空微任务队列后再执行一个宏任务。
常见的微任务
- promise的 then catch finally 
- queueMicrotask 
- new window.MutationObserver(函数); 
常见的宏任务
- 整个script 
- setTimeout
- setInterval
- 交互事件 比如 点击 按键  
- IO操作 读取文件 网络请求回调 

事件循环的一轮是指   
- 宏任务 -> 同步任务 -> 清空微任务队列 -> 判断渲染 -> 判断空闲 —> 下一轮的一个宏任务

## 渲染  
渲染是一个阶段  渲染不是任务也不是队列 是一个阶段， 可以说 渲染这个阶段是异步执行的

## rAF 和 rIC
rAF 和 rIC 既不是宏任务，也不是微任务 ，， 
rAF 是渲染之前 ，同一帧所有的requestAnimationFrame 会合并执行。
rIC 是渲染之后 判断是否有50毫秒的空闲 有空闲就执行rIC  （50ms）    rIC 不建议修改DOM  应用场景 非紧急任务 比如 日志上报 数据预加载

## 执行上下文栈和执行上下文
压栈
出栈

栈顶
栈底

 
- 执行上下文栈 是 JS引擎 使用先进后出的栈结构 来管理 执行上下文。
- 一开始栈底是全局执行上下文。
- 调用函数会创建执行上下文，
- 当前正在执行的上下文 一定在栈顶
- 执行完后就会出栈。

- 执行栈为空或只剩全局执行上下文，事件循环会从任务队列中取出任务，压入执行栈。

- 如果递归没有结束条件，最终就会导致栈溢出。 

## 执行上下文 
执行上下文是 JS引擎 执行代码时创建的运行环境，主要记录了：this指向、词法环境、变量环境。
- 词法环境主要是 保存了 let/const 变量 和 函数声明，以及outer。outer指向父级的词法环境，也就是作用域链。 
- 变量环境主要用于保存 var 变量。 
执行上下文的创建周期： 
函数调用前，会创建函数执行上下文并压入执行栈中，var初始化为undefined 函数初始化为函数对象 是一个引用地址指向堆内存中的实际对象。 let/const没有初始化进入暂时性死区，赋值前访问将报错。
函数执行时，逐行执行代码，给变量更新值。
函数执行结束后，对应的执行上下文从调用栈中弹出。
