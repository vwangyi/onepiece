# 异步编程

JavaScript的异步编程是一种机制，用于解决JavaScript单线程特性与I/O操作（如网络请求、文件读写、定时器等）之间的矛盾，允许程序在执行耗时操作时不会阻塞主线程，从而保证用户界面的响应性。

## 解决异步编程

异步编程的核心机制：
事件循环（Event Loop）：负责协调异步任务的执行。当遇到耗时操作时，JS引擎将任务“挂起”，让其在后台执行，主线程继续处理后续的同步任务。当耗时任务完成后，结果或通知会被放入任务队列（微任务队列和宏任务队列），等待主线程空闲时处理。

异步编程的实现方式：
1.回调函数：最原始的方式，将一个函数作为参数传递给另一个函数，当异步操作完成时调用该回调函数。但多层嵌套会导致“回调地狱”，代码难以维护 。
2.Promise：为解决回调地狱和错误处理问题而出现，代表一个异步操作的最终完成（或失败）及其结果值。提供链式调用和统一的错误处理机制 。
3.async/await：ES2017引入的语法糖，基于Promise构建，使异步代码更像同步代码，提高可读性和可维护性 。
## 应用场景：
网络请求：如fetch API获取数据。
文件读写：在Node.js中读取文件。
定时器：如setTimeout和setInterval。
建议：在实际开发中，优先使用async/await处理异步逻辑，结合Promise的all、race等方法实现并行或竞态需求，回调函数仅用于特定API场景 。
