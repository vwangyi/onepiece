## 手写 call apply bind 方法 
```js 
// 手写 call apply bind方法 
// 都是伪代码 js底层是c++实现的 

// 1 处理第一个参数为null undefined时 指向 globalThis 为原始值时 转为包装类对象 Object('123')
// 2 通过js的规则 this指向调用者 把真实调用函数挂在第一个参数上来调用 实现this指向第一个参数

function call(ctx, ...args) { 
    // 若第一个参数为null或undefined 则为globalThis 若为其他基础类型则转为对象 
    // Object() 传原始型 得到包装类对象
    ctx = [null, void 0].includes(ctx) ? globalThis : Object(ctx); 
    // 处理 调用函数的this指向 第一个形参 
    const fn = this
    const key = Symbol() // 用symbol来防止第一个参数对象 上有相同的属性  
    Object.defineProperty(ctx, key, {
        value: fn,
        enumerable: false
    }) 
    const result = ctx[key](...args) // 利用js的规则 this指向调用者ctx
    return result

}

function apply(ctx, args) { 
    const fn = this
    if(!Array.isArray(args)) {
        throw new Error('the second argument must be an array')
    }
    fn.myCall(ctx, ...args)
}

function bind(ctx, ...args) { 
    const fn = this
    return function (...args2) {
        if(new.target) {
            return new fn(...args, ...args2)
        }
        return fn.apply(ctx, args.concat(args2))
    }
}

Function.prototype.call1 = call;
Function.prototype.apply1 = apply;
Function.prototype.bind1 = bind;
```

## 值传递和引用传递  
- 值传递就是 两个变量完全独立，给其中一个赋值，不影响另一个。
- 引用传递就是 给其中一个变量赋值，另一个也跟着变。
- C++里面有引用传递 JS没有引用传递 全是值传递。
 
## 箭头函数普通函数区别
- 箭头函数没有this，箭头函数使用this是外层的this。
- 箭头函数没有 arguments，批量获取参数 只能使用es6的剩余参数 
- 箭头函数 有 `__proto__` 但没有 `prototype` ，所以不能作为构造函数也不能 new 调用  (Array.prototype.constructor )

## 纯函数
- 纯函数就是 相同的输入始终返回相同的输出，不受外部状态影响，也不修改任何外部状态，没有副作用。

## 闭包
- 闭包就是函数嵌套函数，内部函数可以访问外部函数的作用域。

    ■ 好处
      ● 好处是 可以延长变量生命周期 和 模块化
      ● 当内部函数保持对闭包变量的引用，GC就不能正常回收
      ● 闭包执行，形成私有作用域，不受全局污染，适合模块化开发，commonjs和esmodule都实用了闭包。

    ■ 缺点
      ● 可能导致内存泄露
      ● 不再使用的变量就赋值为null，切断引用让垃圾回收机制回收。
 
  ○ 讲应用
    ■ 闭包的应用场景有 防抖 节流 柯里化 vue中的data函数也是用了闭包

## 递归
- 递归就是函数自己调用自己，注意结束条件就行，不要形成死递归。

## 惰性函数 
- 惰性函数是 第一次调用时 根据实际情况 重写自身，后续调用更高效。这个实际情况后续都不会变。 比如 一些兼容性问题 只需要判断一次就行 不需要每次都判断。

## 防抖和节流 
- 防抖是：在指定时间内，多次触发都会重新计时，只执行最后一次。比如 搜索联想 
- 节流是 多次触发在指定时间内只执行一次。 比如 滚动条频繁滚动触发

## 函数重载
- 函数重载是 可以声明 函数名相同 但 参数个数 或 参数类型不同的同名函数。 js不支持函数重载 只能手动模拟。
- 函数重载：函数重载是 函数名相同，参数的个数或类型不同。

- java语言 可以直接 声明 函数名相同 但 参数个数或参数类型不同的函数。
- js不支持函数重载 但可以手动模拟。
```js 
function createOverload() {
    const map = new Map();

    function overload(...args) {
        const key = args.map(it => typeof it).join(',');
        const fn = map.get(key);
        if (!fn) {
            throw new TypeError('没有找到对应的实现');
        }
        return fn.apply(this, args);
    }

    overload.addImpl = function (...args) {
        const fn = args.pop();
        if (typeof fn !== 'function') {
            throw new TypeError('最后一个参数必须是函数');
        }
        const key = args.join(',');
        map.set(key, fn);
    }
    return overload;
}

const getUsers = createOverload() ;

getUsers.addImpl('xfsfd', () => {
    console.log(23)
})

```
