 
# Nuxt

## 是什么
- Nuxt是基于Vue的全栈框架，可以实现SSR服务端渲染和客户端渲染的混合渲染。




## Nuxt的好处是 
- 缺点：开发需要思考 当前环境是 浏览器还是node ，比如node环境没有window 浏览器环境没有process




## Hydration 注水/激活/水合
- 水合就是 浏览器通过执行js插入DOM 的过程 就是 水合过程




## SSR服务端渲染 / CSR客户端渲染


### 是什么
- 什么是服务端渲染： 服务端返回完整的HTML 
- 什么是客户端渲染： 服务端返回的不是完整的HTML 需要通过执行JS插入DOM 这就是客户端渲染 



## 优缺点
SSR的好处是 
    1. 渲染很快因为不用执行js 
    2. SEO搜索引擎优化友好 
SSR的缺点：
    1. 页面跳转 都需要 重新请求 刷新整个页面。 存在白屏时间 用户体验差。

客户端渲染：
    好处是 页面跳转 交互性强 



交互性强的采用客户端渲染 交互性不强纯浏览采用 服务端渲染，基于这两种的优缺点，我们会采用 首屏是用服务端渲染 次屏采用客户端渲染  

## 实现方案

- SSR 服务端渲染
1. 方案1后端来做：后端Nodejs或Java 直接返回 完整的HTML   

2. 方案2前端来做：使用基于Vue的Nuxt框架 




## 服务端渲染 客户端渲染
- 采用ssr的nuxt框架 交互少 纯浏览 SEO友好 首屏加载快

- 不是所有官网都采用nuxt，只要是 需要SEO 交互少 才使用服务端渲染，
- 如果官网交互性强（和后台管理系统一样），那就采用客户端渲染。

// 典型例子：企业后台管理系统
// - 登录后才可访问
// - 不需要 SEO
// - 复杂的交互和数据表格

// 典型例子：产品官网、企业官网
// - 需要被搜索引擎收录
// - 需要分享到微信/朋友圈显示卡片
// - 首屏加载速度影响转化

如果团队只熟悉 Vue，但又需要 SEO，可以用 

首屏采用服务端渲染 次屏采用客户端渲染 不需要SEO （解决后台管理系统的首屏加载慢）应该采用什么技术 --- 采用 nuxt
