

## mysql的列的数据类型
mysql支持多种类型 大致分为三类 数值 日期/时间 字符串/字符



+ 数值  
tinyint   1字节   很小  
smallint  2字节   小  
mediumint 3字节   中等  
int       4字节   标准  (常用)  
bigint    8字节   大  
float     4字节   单精度浮点数  
double    8字节   双精度浮点数  
decimal   -      字符串浮点数 金融计算常用
+ 字符串  
char      0-255字节  定长字符串  
varchar   0-65535字节 变长字符串 常用  
tinytext  微型文本  2^8-1字节  
text      文本串      2^16-1字节 
+ 日期/时间  
date      YYYY-MM-DD    日期值  
time      HH:MM:SS      时间值  
datetime  YYYY-MM-DD HH:MM:SS   上面两个连起来  最常用  
timestamp 1970.1.1到现在的毫秒数   常用  
year     YYYY          年份值

null 空值

## 表
+ 进入库里面可以对表进行操作

```sql
-- 查看所有表
show tables;

-- 创建表
create table if not exists `表名`( -- 创建一个表 如果不存在
  -- 字段名 数据类型 [长度 默认 主键 非空 Undigned 自增 ZeroFill 更新 注释]

  /**
    * 字段名
    * 数据类型 int(10) 整型长度为10
    * unsigned 表示无符号(正整数)  数据类型为数值 才可以使用
    * not null 表示不为空
    * default '张三' 设置默认值 不为空 一般都要有默认值
    * auto_increment 表示自增
    * primary key 表示主键  primary key(`id`)也可以写在最后一行
    * comment '注释' 表示注释
  */
  `id` int(10) unsigned not null auto_increment primary key comment '主键', 
  `name` varchar(8) not null default '张三' comment '姓名',

  primary key(`id`) -- 每张表必须有主键
  -- engine=InnoDB是数据库引擎 
)engine=InnoDB default charset=utf8mf4 comment='表注释 这是一个用户表';


-- 创建一个用户表
create table if not exists `user`(
  `id` int(10) unsigned not null auto_increment primary key comment '主键',
  `name` varchar(8) not null default '张三' comment '姓名',
  `age` int(3) not null default 0 comment '年龄',
  `sex` enum('男','女') not null default '男' comment '性别',
  `address` varchar(20) not null default '北京' comment '地址',
  `phone` varchar(11) not null default '12345678901' comment '手机号',
  `email` varchar(20) not null default '' comment '邮箱',
  `password` varchar(32) not null default '' comment '密码',
  `create_time` timestamp not null default current_timestamp comment '创建时间',
  `update_time` timestamp not null default current_timestamp on update current_timestamp comment '更新时间',
  `is_delete` tinyint(1) not null default 0 comment '是否删除 0未删除 1已删除'
)engine=InnoDB default charset=utf8mb4 comment='用户表'; 

-- 删表
DROP TABLE 表名;

-- 查看表结构
```

列名 | 数据类型 | 长度 | 默认值 | 主键 | 非空 | 无符号 | 自增 | ZeroFill | 更新 | 注释

unsinged 无符号整数(就是表示正整数 没有负号)

  默认自增1 可以设置自增步长 通常用来设计唯一的主键 index 必须是整数类型

Unsigned   
无符号整数 没有负号的整数

用0填充  
声明该列的长度不够的时候 用0填充  比如 int(3) zerofill  1 -> 001

自增  
自动在上一条记录的基础上 默认 + 1  
通常用来设计唯一的主键  必须是整数类型  
可以设置自增的起始值和步长  auto_increment = 1000 默认 1

非空 （not null）（null）  
设置 not null 该列不能为空  
设置 null 该列可以为空 默认为null 不用设置

## 修改删除表字段
+ 修改

```sql
-- 修改表名 : alter table 旧表名 rename as 新表名;
alert table user rename as user1;

-- 增加表字段 : alter table 表名 add 字段名 字段类型;
alert table user add age int(2) not null default 0 comment '年龄';

-- 修改表字段 : alter table 表名 modify 字段名 字段类型;
alert table user modify age int(3) not null default 0 comment '年龄'; -- 修改约束  modify是修改字段类型
alert table user change 旧名字 新名字 字段类型; -- 修改字段名字 change是修改字段名

-- 删除表字段 : alter table 表名 drop 字段名;
alert table user drop age; 
```

+ 删除

```sql
-- 删除表 尽量加上 判断 if exists
drop table if exists user;
```

## 外键
+ 数据库级别的外健(不用 一般用程序解决 Java在应用层解决)
+ 方式一 创建表的时候就定义外键

```sql
  -- 创建一个班级表
  create table if not exists `class`(
    `id` int(11) not null auto_increment comment '班级id',
    `name` varchar(20) not null unique comment '班级名称',
    primary key(`id`)
  )engine=innodb default charset=utf8 comment '班级表';



  create table if not exists `student`(
    `id` int(11) not null auto_increment,
    `name` varchar(20) not null unique,
    `age` int(11) not null,
    `sex` varchar(10) not null,
    `class_id` int(11) not null,
    primary key(`id`), -- 定义主键
    key `FK_class_id`(`class_id`), -- 把class_id列名作为外健 取名为FK_class_id
    constraint `FK_class_id` foreign key(`class_id`) references `class`(`id`) -- 定义外键
  )engine=innodb default charset=utf8 comment '学生表';

  -- 学生表的 class_id 作为外健 要去 引用 班级表的 作为主健的id字段
  -- 定义外键key
  -- 给这个外键添加约束(执行引用) references引用



  -- 学生表的 class_id 作为外健 要与 班级表的 作为主健的id 关联
  -- 建立关联之后 `student`为主表 `class`为从表
  -- 当一张表 作为另一张表的外健时 一般称为从表
  -- 删除有外键关系的表的时候 必须先删除从表(引用别人的表) 再删除主表(被别人引用的表)

```

+ 方式2 创建表之后再添加外健

```sql
-- 已经有一个班级表 class 和 学生表 student 但没有外健关系
-- 添加外健关系
alert table `student` add constraint `FK_class_id` foreign key(`class_id`) references `class`(`id`);
-- alert table `表名` add constraint `自定外健名` foreign key(`作为外健的列名`) references `引用哪张表的表名`(`引用表的某个列名`);
```

## 外键
> 方式1 在创建表的时候添加外键 增加约束 (麻烦 比较复杂)
>

```sql
create table `grade`(
    -- 
    `gradeid` int(10) not null auto_increment comment '年级id',
    `gradename` varchar(20) not null comment '年级名称',
    primary key(`gradeid`)  -- 设置主键
)engine=innodb default charset=utf8;

-- 学生表的 `gradeid` 字段 要去引用年级表的 `gradeid` 字段
-- 定义外键key
-- 给这个外键添加约束 (执行引用)  references 引用

create table if not exists `student`(
    `id` int(10) not null auto_increment comment '学生id',
    `name` varchar(20) not null comment '学生姓名',
    `age` int(3) not null comment '学生年龄',
    `phone` varchar(11) null comment '电话',
    `address` varchar(50) not null comment '地址',
    `email` varchar(50) not null comment 'email',
    `gradeid` int(10) not null comment '年级id',
    primary key(`id`),
    key `gradeid`(`gradeid`),  -- 定义外键
    constraint `gradeid` foreign key(`gradeid`) references `grade`(`gradeid`)  -- 添加约束
)engine=innodb default charset=utf8;

-- 删除有外键关系的表的时候 必须先删除从表 再删除主表
```

> 方式2 在创建表的时候不添加外键 等表创建完成之后再添加外键
>

```sql
alert table student add constraint `FK_gradeid` foreign key(`gradeid`) references `grade`(`gradeid`);

-- alert table `表名` add constraint 外键名 foreign key(作为外键的字段名) references 哪个表(哪个字段);
```

> 以上的操作 都是物理外键 数据库级别的外键 不建议使用 (避免数据库过多造成困扰)
>

+ 最佳实践


## update 修改。修改表中某条数据 
```sql
-- 修改数据   
-- 跟上条件  
update `表名` set 字段1=值1[,字段n=值n] where 条件;   -- 值可以是变量
update `student` set name='王五',age=20 where id=1;
-- 不跟条件 会匹配所有的数据(所有行) 一般不会这么用
update `student` set name='王五',age=20;

-- 条件: where 子句 运算符 id 等于某个值, 大于某个值, 在某个区间内修改...
-- 操作符会返回 布尔值 true/false

-- 所有的条件表达式 都会返回 布尔值 true/false

操作符               含义      范围      结果
=                   等于      5=6      false
<> 或 !=            不等于     5<>6     true
>
>=
<
<=
between and       在某个区间内  [2, 5]
and               与 我和你     5>1 and 1<2   false
or                或 我或你     5>1 or 1>2    true


-- 通过多个条件定位数据
update `student` set name='王五' where `name`='张三' and sex='女';
-- 条件在什么区间
update `student` set name='王五' where `age` between 18 and 20;  -- age字段 在 18-20 之间

```

## delete 删除  删除表中某条数据 
```sql
-- 删除数据
-- 语法: delete from `表名` [where 条件];

-- delete删除
delete from `student`;  -- 删除所有行(所有数据) 避免这样写 一般不会这样写
delete from `student` where id=1;  -- 删除指定数据

-- truncate清空
-- 作用: 完全清空表中的数据 表结构和索引约束不会变

-- 清空某个表   truncate table `表名`;
truncate table `student`;

-- delete 和 truncate 的区别
-- 相同点: 都是删除数据 都不会删除表结构
-- 不同点: 
  delete from `表名`;  -- 自增不会归零  
  truncate table `表名`; -- 自增会归零 不会影响事务

-- 了解
用 delete 删除然后 重启数据库  有一个现象 
innoDb引擎: 自增列会从1开始 (存在内存当中 断电即失)
myisam引擎: 自增继续从上一个增量开始 (存在磁盘中 断电不会失)
```
